Guida Operativa: Integrazione CI/CD

Progetto: Castello dell'anima (TEI + IA) | File: tassonomia.xml

1️⃣ Verifica la struttura dei file

Il workflow richiede che il repository rispetti una struttura specifica. Assicurati che i seguenti file siano presenti:

  • ✔ tassonomia.xml (nella root)
  • ✔ schema/tassonomia.rng (RelaxNG - richiesto)
  • ✔ schema/tassonomia.sch (Schematron - richiesto)
⚠️ Attenzione ai file Schema Il documento XML punta a schema/tassonomia.rng. Se questi file non sono presenti nella cartella schema/, il workflow segnalerà un errore al passaggio di validazione. Crea la directory o adatta il file main.yml di conseguenza.

2️⃣ Inserisci il Workflow

Crea la directory standard per GitHub Actions e posiziona il file di configurazione:

Terminale
mkdir -p .github/workflows touch .github/workflows/main.yml

3️⃣ Cosa verifica lo script?

Il workflow è configurato per proteggere le 8 tassonomie core del progetto:

func relation impact risk mystic_state operation exposition phase

Se uno di questi xml:id risulta assente nel file XML, la pipeline interromperà l'esecuzione (status ❌).

4️⃣ Esecuzione e Deploy

Dopo aver salvato i file, esegui il commit e il push. Il sistema si attiverà automaticamente.

Git
git add . git commit -m "chore: aggiungi validazione CI per tassonomia TEI" git push origin main

Monitora l'avanzamento dalla scheda Actions del repository GitHub.

5️⃣ Test in Locale (Opzionale)

Per validare localmente il file prima del push (richiede xmllint):

Locale
# Verifica sintassi XML xmllint --noout tassonomia.xml # Verifica rispetto allo schema RelaxNG xmllint --noout --relaxng schema/tassonomia.rng tassonomia.xml
🔗 Chiusura del ciclo normativo: Questo workflow applica automaticamente i vincoli definiti nel documento normativo “Sistema Tassonomico.md”, garantendo allineamento continuo tra documentazione, schema e repository (README → schema → CI).
